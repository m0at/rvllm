23 Rust crates. 50 hand-written CUDA kernels. 792 tests. built in ~22 hours with Claude Opus doing architecture, CUDA kernels, debugging, and 30-agent swarms for major optimization passes. $1,600 vast.ai GPU rentals + $280 Claude Code usage.

what changed since last post
- FA3 v3 attention: cp.async bulk global-to-shared copies (128-bit, bypasses registers/L1) and split-KV that distributes KV tiles across thread blocks. 31-42% faster than FA3 v2 across all concurrency levels.
- killed all silent kernel fallbacks. startup now validates every kernel loads or the server crashes. was masking missing fused kernels and silently degrading to slow paths.
- JIT PTX compiler: Rust emits shape-specialized fused kernels at model load. Add+RMSNorm+GateUp GEMV 5.12x faster, SiLU*Mul+Down GEMV 7.48x faster than the hand-written CUDA equivalents. no nvcc, no Triton, no Python.
- CUTLASS header integration for sm_90 attention primitives.
- found and fixed the coherence bug. fused QKV GEMM outputs [T, QKV_dim] (interleaved per token) but the split assumed [all_Q | all_K | all_V] layout. identical for T=1 (decode), completely wrong for T>1 (prefill). corrupted the KV cache on every prompt. fix: unfused projections for prefill, fused for decode.
- rms_norm_eps was hardcoded at 1e-5 instead of reading from model config (Qwen2.5 uses 1e-6). now read from config.json and logged at startup.
- pure f16 data path. f32 only remains for RoPE trig tables and the final argmax logits boundary.
- pre-allocated metadata + output GPU buffers at max size so CUDA graph-captured pointers never go stale on reallocation.
- removed ~530 lines of dead f32 CPU attention fallback code and duplicate f32 weight storage.

architecture
- pure f16 forward: embed(f16) -> 28 layers(f16) -> norm(f16) -> LM head(f16xf16->f32) -> argmax
- three-tier kernel stack: JIT fused (2-7.5x faster) > 50 hand-written CUDA > cuBLAS autotuned
- fused QKV + gate+up weights for decode (T=1): 5 GEMMs -> 2 per layer
- unfused for prefill (T>1): correct [all_Q | all_K | all_V] layout
- cross-layer residual+RMSNorm fusion
- FA3 v3 decode attention: cp.async, split-KV, parallel softmax, bank-conflict-free smem, GQA V reuse
- CUDA graph capture per exact batch size, 1 cuGraphLaunch replaces 284 kernel launches
- custom GEMV (M=1) + cublasLt split-K (M<=32) + cuBLAS hgemm (M>32)
- no-fallback kernel validation at startup -- every kernel loads or server exits
- dedicated GPU OS thread, async DtoH with pinned memory
- 10 architectures: Llama, Mistral, Qwen2, Phi, Gemma, Mixtral, DeepSeek, GPT-NeoX, StableLM, Cohere
