1k-token correctness sample

This is the actual PPL sanity check for the current StableHLO path, not the old Mosaic body numbers and not a sampled reply. PPL is a teacher-forced next-token metric by definition: feed token t, score token t+1. The answer path can still produce an up-to-1k-token reply, but this check verifies the PPL is not secretly low yet.

Run Tokens scored Avg NLL PPL Top-1 Status
StableHLO B=1 correctness sample 1,000 142.235 5.92e61 0.0% PPL bad, expected
The 1k-token sample confirms the model is not at real language-model correctness yet: PPL is extremely high and top-1 next-token match is zero. That does not contradict the up-to-1k reply smoke working as an execution path; it means the generated answer should not be treated as model-quality evidence until MoE is wired and PPL comes down.

Path to real PPL

The active path is pure StableHLO inline: keep the graph in PJRT-native StableHLO, add the MoE block, then split KV so B=32 no longer needs a single >2^31 element tensor.

Path Approach Est. hours PPL target
Pure StableHLO inline Add native MoE router + expert path, keep dense weights as bf16 input arena 8-16 ~4.9
Split KV layout Shard KV by layer or device-visible chunks so B=32 avoids >32-bit HLO shape indices 4-8 ~4.9
JAX reference parity Compare per-layer hidden states after attention and after MoE against JAX 16-24 ~4.9
Current blocker: B=32 compile fails after emit because XLA refuses a dynamic-update-slice on the 8.32 GB int8 KV tensor when shape indices exceed 32 bits. B=1, B=8, and B=16 execute end to end.
