Path to real PPL

Three paths were evaluated. The plan is to extract the JAX layer, lower to StableHLO, and link as a Mosaic body.

Path Approach Est. hours PPL target
JAX-extract body JIT-lower JAX layer to StableHLO, serialize via mosaic-serde, link into rvLLM graph 8-16 ~4.9
Pure StableHLO inline Emit full layer math as inline StableHLO ops in Rust (no Mosaic body) 12-16 ~4.9
Pallas kernel Write M2 layer as pl.pallas_call, lower via JAX, produces valid Mosaic bytecode 16-24 ~4.9
Blocker found: JAX-emitted StableHLO bytecode was rejected at the Mosaic body slot -- PJRT enforces stable_mosaic.version attrs that JAX lowering doesn't produce. The current plan wraps the full JAX decode graph (not just the body) so rvLLM consumes it as a complete StableHLO program via PJRT.
